# @asochan-ai Skills — はじめてのガイド

## これは何？

Claude Code（AIコーディングツール）に**追加能力**を入れるスキルパック。
「設計してから作る」ワークフローを、AIが自動でやってくれるようになる。

## 何が嬉しいの？

普通にClaude Codeを使うと:
```
あなた: 「〇〇作って」
Claude: （いきなりコードを書き始める）
→ 途中で「あ、これ考えてなかった」→ 手戻り → 時間の無駄
```

このスキルパックを入れると:
```
あなた: 「〇〇作って」
Claude:
  1. まず何を作るか確認（スコープ定義）
  2. 既存コードと外部情報を調査（リサーチ）
  3. 設計書を書いてあなたに見せる（承認ゲート）
  4. OKもらってから実装開始
→ 手戻りが激減。最初から正しい方向に進む
```

```
あなたの指示
    │
    ▼
┌─────────────────────────────────────────────────┐
│  super-plan（設計フェーズ）                         │
│                                                   │
│  Step 1: 「何を作る？完了条件は？」を明確化          │
│  Step 2: 4つのAIエージェントが並列でリサーチ         │
│     ┌──────────┬──────────┬──────────┐            │
│     │ Agent A  │ Agent B  │ Agent C  │ Agent D   │
│     │ 既存コード│ 過去の   │ 外部の   │ 受け手に  │
│     │ 調査     │ 教訓調査 │ 技術調査 │ 最適な    │
│     │          │          │          │ 設計調査  │
│     └──────────┴──────────┴──────────┘            │
│  Step 3: 設計書を自動生成                           │
│  Step 4: エビデンスチェック（推測を許さない）         │
│                                                   │
│  ★ あなたが設計書を確認 → 「OK」or「ここ変えて」    │

┌─────────────────────────────────────────────────┐
│  rapid-build（実装フェーズ）                        │
│                                                   │

└──────────────────┬──────────────────────────────┘

                   ▼

│  security-audit → セキュリティ監査                   │
│  verify        → 最終チェック                       │
└─────────────────────────────────────────────────┘
```

## インストール方法

### Claude Code（CLI）を使っている場合

```bash
# 1. マーケットプレースを登録（1回だけ）
/plugin marketplace add asochan-ai/claude-plugins

# 2. おすすめセット（全部入れても軽い）
/plugin install super-plan@asochan-ai
/plugin install rapid-build@asochan-ai
/plugin install pre-check@asochan-ai
/plugin install quality-rules@asochan-ai
/plugin install auto-test@asochan-ai
/plugin install security-audit@asochan-ai
/plugin install verify@asochan-ai
```

### Claude Cowork（GUI）を使っている場合

1. Claude Desktop App を開く
2. 「Cowork」タブへ
3. Plugin検索で「asochan-ai」を検索
4. インストール

（※ 公式Marketplace提出後に利用可能になります）

## 使い方

### 基本: super-plan を呼ぶだけ

```
あなた: 「ユーザー登録機能を作りたい。super-planで設計して」
```

これだけで:
1. スコープを確認する質問が来る
2. 4エージェントが並列リサーチ
3. 設計書が生成される
4. 「承認しますか？」と聞かれる
5. 承認後、rapid-buildで実装開始

### おすすめの使い方

| やりたいこと | 言い方 |
|-------------|--------|
| 新機能の設計 | 「super-planで〇〇を設計して」 |
| 設計→実装まで一気に | 「super-planで設計して、承認したらrapid-buildで実装して」 |
| セキュリティチェック | 「security-auditして」 |
| テスト実行 | 「テスト走らせて」 |
| プレゼン資料作成 | 「〇〇の提案スライド作って」 |

### 使わなくていい場面

- ちょっとした修正（1ファイルの変更）→ 普通にClaude Codeに頼めばOK
- 調査だけ（コード変更なし）→ 普通に聞けばOK
- 既に設計が決まっている → rapid-buildだけ使う

## スキル一覧と役割

### メインスキル（この2つを覚えればOK）

| スキル | 一言で言うと |
|--------|-------------|
| **super-plan** | 「作る前に調べて設計書を書く」|
| **rapid-build** | 「設計書通りに作る」 |

### 品質ガード（入れておけば自動で動く）

| スキル | 一言で言うと |
|--------|-------------|
| **pre-check** | 作業前に「何に影響するか」を教えてくれる |
| **quality-rules** | コードを書く時に品質ルールを守ってくれる |
| **auto-test** | テストフレームワークを自動検出して実行 |
| **security-audit** | セキュリティの穴を3段階でチェック |
| **verify** | 全体の品質を一括チェック |

### コンテンツ作成

| スキル | 一言で言うと |
|--------|-------------|
| **presentation-architect** | スライド/提案書を自動生成 |
| **frontend-slides** | HTMLスライドのエンジン（architectが呼ぶ） |
| **svg-diagram** | 図解をSVGで生成 |

### 並列処理

| スキル | 一言で言うと |
|--------|-------------|
| **multi-agent** | 大量タスクを複数AIで並列処理 |

## よくある質問

**Q: Claude Codeって何？**
A: Anthropicが作ったAIコーディングツール（CLI）。ターミナルで動く。https://claude.ai/download

**Q: 無料？**
A: このスキルパックは無料（MIT License）。Claude Code自体はAnthropicのAPI料金が必要。

**Q: 他のAIツール（Cursor, Codex等）でも使える？**
A: はい。SKILL.mdはオープンスタンダードで、35+プラットフォームで動作します。

**Q: 入れたら重くなる？**
A: いいえ。スキルはMarkdownファイルなので、必要な時だけ読み込まれます。

**Q: super-plan無しでrapid-buildだけ使える？**
A: はい。既に設計書（PLAN_*.md）があれば、rapid-buildに直接渡せます。
